# Deficiency Register

<!-- NUCLEAR-GRADE-PLACEHOLDER: replace every field below with real content, then delete this line so validation can pass. -->

**Purpose:** Keep known problems aged, owned, and either fixed or formally risk-accepted, so deviations are not quietly normalized into the new standard.

**Activation threshold:** Use when a known problem will outlive a single change — a flaky test, a noisy alert, an unowned service, a dead dashboard, a recurring incident, or deferred hardening.

**Minimum useful version:** for each deficiency, the description, the date first seen, the owner, the disposition (fix-by-date or formal risk-acceptance), and the review trigger.

## Exit criteria

- Every entry has an owner and a disposition; none sit as "known, untracked."
- Accepted risks carry a named owner and a concrete revisit trigger.
- The age of each deficiency is visible so old accepted risks resurface for review.
