# Adversarial Eval — New Skills (2026-06-26)

Method (per skill-creator): for each skill, 5 adversarial cases across the
categories — **A** ambiguous input · **B** boundary overlap · **C** wrong-domain
adjacent · **D** regression (should fire) · **E** edge case. Each case is judged
on whether the skill's `description` + `auto-trigger`/`do-not-trigger` routes
correctly (fire / defer). `pass_rate` = correct routings ÷ 5. `trigger_accuracy`
= same denominator, tracking false-fire vs false-defer separately in notes.

Scope: the 12 skills added this session (PRs #40, #49, #50).
