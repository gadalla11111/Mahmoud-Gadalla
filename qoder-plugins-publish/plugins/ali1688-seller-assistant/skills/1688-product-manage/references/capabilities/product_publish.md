# 商品发布

## 功能

`product_publish` 支持三种发品模式：

| 模式 | 说明 |
|------|------|
| 本地图片上传 | 文件验证 → 图片压缩（≤1MB）→ 上传图片银行 → 获取图片 URL |
| 图片链接发品 | 直接提供图片 URL，跳过上传 |
| 批量发品 | 多张图片 → 批量上传 + 批量 AI 识图 → 汇总选品 → 生成发品 Excel |

---

## 单图发品流程

```
获取图片URL → AI识图 → ⏸ 等商家选品 → 保存选品 → 生成跳转链接 → ⏸ 流程结束
```

**步骤 1 — 获取图片 URL**
- 本地图片：上传至图片银行，返回图片 URL
- 图片链接：直接使用用户提供的 URL
- CLI：`product_publish --image <path>` 或 `--url <url>`

**步骤 2 — AI 识别同款商品**
- 将图片 URL 提交 AI 识图，返回匹配的商品类目和同款商品列表
- 此步自动完成，无需单独调用

**步骤 3 — 展示同款，等待商家选择（⛔ 必须停下）**
- 展示同款商品列表，使用序号（1、2、3...）
- ✅ 标题、属性必须展示（`sameItemSellerPoint` 为属性键值对形式）；主图视对话空间决定，若展示须用 `![同款N](url)` 渲染
- ⛔ 即使只有 1 个候选，也必须等商家明确选择后才能继续

**步骤 4 — 保存选品数据**
- CLI：`product_publish --select <N>`（N 为商家选择的同款序号）
- 返回 `aigcSelectCategoryTime`（UUID）

**步骤 5 — 生成发品跳转链接（流程结束）**
- 根据 `aigcSelectCategoryTime` 和 `categoryId` 拼接跳转链接
- ⛔ **链接展示后流程即结束**，不得轮询；商家在 1688 页面补充信息后自行发布
- 同时提示：发布后可用 AI 标题优化、AI 主图优化进一步提升商品质量

**可用 API：** [图片银行上传](../apis/image_bank_upload_picture.md) · [AI 发品识别](../apis/ai_publish_by_imageurl.md) · [保存选品](../apis/ai_publish_save_tair.md)

## 批量发品流程

```
批量获取URL → 批量AI识图 → 紧凑展示同款 → ⏸ 等商家选方式 → [选项2? → HTML勾选 → ⏸ 等粘贴] → 生成Excel → ⏸ 流程结束
```

**步骤 1 — 批量获取图片 URL**
- 本地图片逐张上传至图片银行；单张失败不中断，继续处理剩余图片
- 图片链接直接使用
- CLI：`product_publish --images <p1> <p2> ...` 或 `--urls <u1> <u2> ...`

**步骤 2 — 批量 AI 识图**
- 对每个图片 URL 进行 AI 识图，每张图返回各自的同款候选列表
- AI 无法识别或未返回同款的图片，自动过滤，不展示且不生成发品链接

**步骤 3 — 紧凑展示同款候选（⛔ 必须停下）**
- 仅展示有同款的图片，每张只展示首选同款（第 1 个候选）
- ✅ 标题、属性必须展示；主图视对话空间决定，若展示须用 `![同款N](url)` 渲染
- 同时告知每张图片的同款总数（如「同款数：4」）
- 自动保存批量识图状态到 `.batch_publish_state.json`
- ⛔ 必须提供两个选项等商家选择：
  - **选项 1：默认全选第一个同款** → 直接进入步骤 5 生成 Excel
  - **选项 2：查看所有同款后再选择** → 进入步骤 4

**步骤 4 — HTML 勾选界面（商家选了选项 2 时进入）**
- CLI：`product_publish --show-all-candidates`
- 系统自动生成 HTML 勾选界面并在浏览器中打开，Agent 不得再调 Python 重复生成
- 按图片顺序逐张展示所有同款候选，每项必须同时展示：**主图** `![同款N](url)` + **标题** + **属性**，缺一不可
- 商家在 HTML 中勾选后复制结果文本，粘贴回聊天框
- ⛔ 等商家粘贴结果后才可继续，严禁 Agent 自行选品
- HTML 协议详见 [`publish_select_same_html.md`](../apis/publish_select_same_html.md)

**步骤 5 — 批量保存选品 + 生成发品 Excel**
- 选项 1 路径：`product_publish --batch-default-first`
- 选项 2 路径：`product_publish --batch-select "1,2,1"`（逗号分隔每张图的同款序号）
- 自定义选品序号数量必须与有同款的图片数量一致
- Excel 输出规范：

  | 项目 | 规范 |
  |------|------|
  | 命名格式 | `批量发品链接_YYYY年M月D日HH:MM_过期YYYY年M月D日HH:MM.xlsx` |
  | 表头 | `序号` \| `本地图片`（120×120 px 缩略图，仅展示图片，失败则留白） \| `同款标题/发品链接`（超链接） \| `同款属性` |
  | 有效期 | 3 天（文件名已标注过期时间，对话中须提示商家及时发布） |

- ⛔ 所有链接生成后流程即结束，不得轮询

**可用 API：** 同单图

## 禁止事项

| 规则 | 说明 |
|------|------|
| 禁止暴露内部 ID | 展示同款时严禁暴露 sameItemId、categoryId、tkItemIds 等 |
| 禁止跳过选品确认 | 任何模式下都必须等商家明确选择，即使只有 1 个候选也不得自动选品 |
| 禁止 Agent 自行选品 | 不得以"高度相似"等理由代替商家做选品决策，违反将导致商家发布错误商品 |
| 禁止纯文本 URL | 主图须用 `![alt](url)` 渲染，不得只贴纯文本 URL |
| 禁止轮询发布结果 | 链接生成后流程结束，不得轮询 |
| 禁止重复生成 HTML | 步骤 4 的 HTML 由系统自动生成，Agent 不得再调 Python 生成 |
| 禁止图片上传到非图片银行 | 所有图片只能上传至图片银行（cbu01.alicdn.com），对外 URL 须确保来源为图片银行或合法公开 URL |
| 禁止混用单图/批量参数 | 批量与单图模式参数组不可混用 |

## 异常处理

| 错误 | 原因 | 解决 |
|------|------|------|
| 文件不存在 | 图片路径错误 | 确认路径正确 |
| 图片压缩失败 | 图片格式不支持 | 使用 JPG/PNG 格式 |
| AK 未配置 | 未配置认证信息 | 配置 AK 认证信息 |
| AI 无法识别 | 图片无法匹配商品 | 跳转手动发品页面 `https://offer-new.1688.com/select.htm` |

**图片压缩策略**：Pillow 库，先逐步降低 JPEG quality（从 95 起每次 -5），仍超限则缩小分辨率（每次 ×0.8），最多 5 轮。支持 JPG、PNG、WEBP 格式。

**批量状态文件**：识图结果保存在 `.batch_publish_state.json`，每次批量识图覆盖上一次；后续查看候选、选品均从该文件读取。与单图状态独立。
