# Prompt 编辑器 HTML

**生成方式**: `customize` 阶段自动生成并在浏览器中打开（场景已由 Agent 推荐注入后）。**HTML 是首次 customize 的必要产物，商家可视化编辑「图片比例 / 文字卖点 / 场景」的核心入口，必须自动生成且自动打开，无例外**。商家选"调整"后重跑 customize 时不再生成 HTML。  
**输出路径**: `scripts/prompt_editor_{offer_id}.html`  
**用途**: 为商家提供可视化的提示词编辑界面，**允许调整「图片比例 / 文字卖点 / 场景」**，提示词由系统按规则自动拼装（只读展示），编辑完成后一键复制带入聊天框。

## 触发时机

在 `ai_image_improve --offer_id <X> --customize <JSON>` 执行结束时自动生成并打开。

> 注意：`prepare` 阶段**不再**生成 HTML。HTML 在 customize 阶段才打开，确保表格中的「场景」列已带 Agent 推荐内容，避免商家面对空场景。

> ℹ️ **重跑 customize 规则**——当商家选择"调整"后，Agent 用新参数重跑 customize 时，**不得再次生成 HTML 文件**，仅更新状态并在聊天框渲染新的 prompt 表。HTML 仅在首次 customize 时生成一次。

## 表格结构

| 列名 | 类型 | 说明 |
|------|------|------|
| 序号 | 只读 | 主图位次，从 1 起递增 |
| 主图 | 只读 | 原图缩略图（80×80），来源于 prepare 拉取的主图 URL |
| 图片比例 | 下拉选择 | `1:1` / `3:4`；修改后按默认规则重新生成「提示词」 |
| 文字卖点（可编辑、可选填） | 文本框 | **默认为空、无数据来源支撑，仅可由商家主动填写**；例：`“双头设计”和“油性不晕染”`，修改后按默认规则重新生成「提示词」 |
| 场景（可编辑） | 文本框 | 初始值为 Agent 通过 `--customize` 传入的 `background`；修改后按默认规则重新生成「提示词」 |
| 提示词（**只读展示**） | 灰底 div | 由系统按 `比例 + (可选)文字卖点 + (可选)场景` 自动拼装；**不支持手动编辑**，避免商家随意输入非规范提示词 |
| 操作 | 按钮 | 单个行级按钮（见下方） |

> ⛔ 「提示词」列仅为只读展示，仅由 `syncPrompt()` 在比例 / 文字卖点 / 场景变化时刷新；代码中不读取 `selling_points`（图像可见卖点）字段，**也不接受商家手填整句 prompt**。

## Prompt 拼装规则（默认规则，唯一来源）

商家无法手动覆盖提示词；任何对「比例 / 文字卖点 / 场景」的修改都会触发该规则重新生成提示词。规则与 CLI [`_build_prompt`](../../scripts/capabilities/ai_image_improve/cmd.py) 完全一致：

- **仅比例**：`做一张1688商品电商的主图,{size}比例`
- **比例+场景**：`做一张1688商品电商的主图,{size}比例，背景换成{scene}`
- **比例+文字卖点**：`做一张1688商品电商的主图,{size}比例，突出{tsp}的文字卖点`
- **比例+文字卖点+场景**：`做一张1688商品电商的主图,{size}比例，突出{tsp}的文字卖点,背景换成{scene}`

示例：`做一张1688商品电商的主图,1:1比例，突出“双头设计”和“油性不晕染”的文字卖点,背景换成米白色大理石台面 + 暖色自然光`

> 标点说明：「比例」后使用中文逗号「，」接「文字卖点」或「场景」；「文字卖点」后接「场景」时使用英文逗号「,」。

## customize 入参字段白名单（CLI 强校验）

CLI 的 [`_validate_customize_keys`](../../scripts/capabilities/ai_image_improve/cmd.py) 对 `--customize` JSON 进行字段白名单校验：

- ✅ 允许：`size`（图片比例，仅 `1:1` / `3:4`）、`background`（图片场景，可选）、`text_selling_points`（文字卖点原文，可选、默认空，仅可由商家主动填写）
- ⛔ 拒绝：`prompt`、`selling_points`（图像解析卖点） 或其他任何字段 → 报错 `INVALID_PARAM`

报错信息：
```
{where} 出现不允许的字段：[...]。仅支持 'size'（图片比例）、'background'（图片场景）、
'text_selling_points'（文字卖点，商家手填原文，可选）；提示词由系统按规则自动拼装，
**不允许提交 'prompt' / 'selling_points' 等自定义字段**。
```

> 🔔 **`text_selling_points` 与历史 `selling_points` 的区别**：历史的 `selling_points` 是「图像解析可见卖点」（已禁用）；新增的 `text_selling_points` 是「商家手填的文字卖点原文」，拼接后为「突出{tsp}的文字卖点」。Agent 不得从商品信息 / 图像解析中推断填充该字段。

## 交互按钮

所有按钮均为「**复制到剪贴板**」行为，**不会修改表格内容**。按钮仅做剪贴板复制 + Toast 反馈，不引导跳转。

### 行级按钮（每行一个）

| 按钮文案 | 功能 |
|------|------|
| 📋 用此行覆盖全部图 | 把该行「提示词」作为全部图的提示词，按 `图{n}: {该行 prompt}` 格式拼接并复制到剪贴板（不修改表格中其他行的显示）；Toast 提示「✅ 已复制（第 N 行作为全部图提示词）」 |

### 全局按钮（表格上方一个，为主要 CTA）

| 按钮文案 | 功能 |
|------|------|
| 📋 一键复制全部提示词 | 把每行各自的「提示词」按 `图{n}: {prompt}` 格式拼接并复制到剪贴板；Toast 提示「✅ 已复制全部提示词」 |

## 复制到剪贴板的格式

```
图1: 做一张1688商品电商的主图,1:1比例，突出“双头设计”和“油性不晕染”的文字卖点,背景换成米白色大理石台面 + 暖色自然光
图2: 做一张1688商品电商的主图,1:1比例
图3: 做一张1688商品电商的主图,3:4比例，背景换成原木桌面 + 浅灰背景 + 绿植点缀
```

商家复制后粘贴到聊天框，Agent 基于此内容判断是否需要再次 `--customize`，确认后进入 generate。

> ℹ️ **商家从 HTML 复制提示词粘贴回 Agent**——剪贴板格式为「图N: 完整 prompt 句子」逐图覆盖全图。Agent 收到后用商家给的参数重跑 customize，此时**不生成 HTML**，仅更新状态并渲染 prompt 表，然后回到回合 ② 询问商家确认。


## 使用引导（HTML 顶部黄色提示条）

> 💡 **使用说明**：你可调整「图片比例」「文字卖点」「场景」三个字段，**「文字卖点」默认为空、无数据来源，由你自行选填**（填完拼接为“突出{你填写的文字卖点}的文字卖点”）；**「提示词」由系统按规则自动生成，不支持手动编辑**。修改任一字段后，对应行的「提示词」会自动刷新。**调整完后点击下方按钮即可复制提示词**。

### ⛔ 严禁的替代行为

customize 执行后，Agent **严禁采用以下任何方式替代 HTML 编辑器收集参数**：

| 序号 | 违规行为 | 示例 |
|------|---------|------|
| 1 | 在聊天框中直接询问全部参数 | ❌「请告诉我你想要的图片比例、卖点和场景」 |
| 2 | 逐图分别追问参数 | ❌「图 1 的背景想换成什么？」「图 2 的卖点是？」 |
| 3 | 表单式对话收集 | ❌ 列出字段名让商家逐一填写 |
| 4 | 跳过 HTML 直接调用 generate | ❌ 未经 HTML 编辑器确认就拼装 prompt 执行生成 |

> **唯一例外**：商家主动在聊天中粘贴「图N: 完整 prompt」重跑 customize 时传 `--no-html`（直通 generate 路径），但**首次 customize 绝不允许跳过 HTML**。

## Agent 聊天框输出规范（强约束）

### 常规回合②（未传 `--no-html`）

customize 执行后，Agent **必须在聊天框「最终 prompt 表」上方**醒目提示商家网页已打开，引导其去网页调整提示词。输出需包含：

1. **加粗主提示**（带 ✨ emoji）：「已为您打开《AI 主图优化 - 提示词编辑器》网页」，明确告知可在网页调整**图片比例 / 文字卖点 / 场景**
2. **四条操作引导**（项目符列表）：
   - 调整「图片比例」「文字卖点」或「场景」→ 对应行的「提示词」会按默认规则自动重新生成
   - 「文字卖点」默认为空、无数据源支撑，可由您自行选填（例：`“双头设计”和“油性不晕染”`）
   - 提示词由系统按规则自动生成，**不支持手动编辑**（也不接受 customize 注入 `prompt` / `selling_points` 字段）
   - 调整完后点击「📋 一键复制全部提示词」复制每张图各自的提示词；或点击行级「📋 用此行覆盖全部图」把当前行提示词作为全部图的提示词复制。
3. **手动打开兜底**（引用格式）：「网页未自动打开？手动打开文件：`scripts/prompt_editor_{offer_id}.html`」
4. 以上提示与表格之间用 `---` 分隔，后面接「### 当前 Prompt 表」小标题再展示表格

⛔ 禁止：把「网页已打开」提示放在表格下方、省略不发、或合并到表格表头句中。

### 直通 generate 例外路径（传了 `--no-html`）

商家以「图N: …」逐图结构化文本覆盖全图、Agent 重跑 customize 且传了 `--no-html` 时：

1. **不生成/不打开 HTML**——CLI 跳过 `_generate_prompt_html` 调用
2. **Agent 聊天框不得输出**上述「✨ 已打开网页」主提示、「四条操作引导」、「手动打开兜底」三段内容（CLI 也不输出）。CLI 会改为输出一行「🚀 直通 generate 路径」提示以示路径选择
3. **仍须渲染「当前 Prompt 表」**（保留完整 prompt 独立列），供商家最后核对
4. 走完 customize 后直接进入 generate，不再询问回合③两选项

## 文件生命周期

- **生成时机**: 每次执行 customize 时覆盖写入
- **清理策略**: 不自动清理（HTML 为静态参考文件，不影响流程状态）
- **状态独立**: HTML 仅供商家可视化编辑，不写入 `.image_state.json`，实际 prompt 以系统按规则生成的为准

## 技术说明

- 纯前端 HTML + JavaScript，无后端依赖
- 使用 `navigator.clipboard.writeText` 实现剪贴板复制（降级为 `execCommand('copy')`）
- 生成后通过系统命令自动打开（macOS: `open` / Windows: `startfile` / Linux: `xdg-open`）
- 提示词列使用 `<div class="prompt-display">` 只读展示，`syncPrompt()` 通过 `.textContent` 直接刷新
- 行级与全局按钮均为「只复制到剪贴板」行为，不修改 `ROWS` 数据与表格 DOM
- ⚠️ Python 源码中所有 emoji 必须使用真实字符（如 `📋`），**禁止使用 UTF-16 代理对转义**（如 `\ud83d\udccb`），否则会触发 `surrogates not allowed` UTF-8 编码错误
