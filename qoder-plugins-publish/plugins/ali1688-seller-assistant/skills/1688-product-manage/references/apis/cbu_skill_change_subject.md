# 修改商品标题

**接口路径**: `POST /api/cbu_skill_change_subject/1.0.0`  
**用途**: 修改指定商品的标题

## 请求参数

| 字段名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| offerId | string | 是 | 商品 ID |
| subject | string | 是 | 新标题（受长度规则约束，详见补充说明） |

## 响应结构

| 字段名 | 类型 | 描述 |
|--------|------|------|
| success | boolean | 标题是否修改成功 |

## 示例

### 请求示例

```json
{
  "offerId": "728458413589",
  "subject": "2024新款女士单肩包真皮手提包简约通勤"
}
```

### 响应示例

```json
{
  "success": true
}
```

## 补充说明

### 标题长度规则

- 中文字符（含中文标点）按 **1 字符** 计算
- 英文字母、数字、半角符号按 **0.5 字符** 计算
- 折算后总长度 **不得超过 30 字符**
- 超长标题需先在调用方裁剪 / 提示商家修改，避免接口校验失败

### 权限与前置条件

- 调用前必须完成 AK 配置（`scripts/authorize.py --mode AK`）
- 仅可修改 **当前账号名下的商品**，跨账号或非自有商品会被拒绝
- offerId 必须真实存在且处于可编辑状态（已下架 / 锁定状态可能不允许改标）

### 错误场景

| 场景 | 表现 | 处理建议 |
|------|------|----------|
| offerId 无效 | success = false | 通过 [商品基础信息查询](./cbu_offer_query_tool.md) 校验 offerId |
| 标题超长 | 接口报错 / success = false | 按上述长度规则重新生成标题 |
| 标题命中违禁词 | 接口报错 | 移除违禁词后重试 |
| AK 未配置或失效 | 鉴权失败 | 重新执行 `configure` 配置 AK |
