# Tech Service Suite

> A full-process tech service tool suite that connects both the demand and supply sides into a closed-loop workflow. The demand side covers tech demand mining, achievement search & matching, valuation, deal plan generation, full-lifecycle project tracking, and policy & funding matching. The supply side covers product opportunity discovery, product definition & planning, technical solution demonstration, prototype validation, mass production planning, commercialization strategy, and fundraising preparation. Supports automatic enhancement when connected to Yiqixin, Zheqi Home, and other services, automatically linking to eight major categories of national tech service resources: tech finance, comprehensive tech consulting, enterprise incubation, intellectual property, R&D design, technology transfer and commercialization, inspection and testing, information technology, and intellectual property. Can also be used with pure manual input.

## Who Is This For

### Demand Side (Technology Acquirer / Technology Manager)


| Role | What Problem It Solves |
| ----------------------------- | ---------------------------------------------------------------- |
| **Technology Manager** | Systematically manage the full process of technology achievement commercialization, improving service standardization and professional output quality |
| **Tech Service Institution** | Standardize achievement evaluation and deal-making processes, shortening project cycles |
| **University / Research Institute Tech Transfer Staff** | Standardize achievement evaluation, valuation, and deal-making processes |
| **Enterprise Technology Strategy Department** | Assist technology acquisition decisions and policy applications |

### Supply Side (Technology Provider / Tech Entrepreneur)


| Role | What Problem It Solves |
| ----------------- | ------------------------------------------ |
| **Tech Entrepreneur** | Systematically complete full-process planning from opportunity identification to product launch |
| **Product Manager** | Standardize product definition, validation, and commercialization strategy |
| **Hardware Engineer** | Structurally complete EVT/DVT/PVT validation and mass production preparation |
| **Investor / Incubator** | Assess R&D maturity and commercialization feasibility of startup projects |

## When Not to Use

- Scenarios requiring substitution for professional legal, financial, or technical evaluation opinions
- Transfer evaluations involving state secrets or classified technologies
- Technologies subject to export control lists in cross-border technology transfer
- Fundraising outputs do not constitute securities issuance, investment solicitation, or FA services
- Market forecasts and valuations are hypothetical estimates only and do not substitute for professional judgment

## Workflow

### Demand Side — Full Technology Introduction Process

```
Step 1: /tech-demand-mining
  Input: Enterprise name or industry keyword
  Output: Demand profile (pain point ranking, discipline mapping), technology demand list, recommended docking resources

Step 2: /achievement-search-matching
  Input: Technology demand description + maturity expectation
  Output: Ranked list of recommended technology achievements/patents/teams + technology route comparison

Step 3: /technology-valuation
  Input: Patent number or technical solution description
  Output: Commercialization feasibility assessment report (advancement, market space, risks, pricing reference)

Step 4: /deal-plan-generation
  Input: Supplier achievement info + demander enterprise info
  Output: Technology transfer cooperation proposal (model, transaction structure, milestones, risk control)

Step 5: /project-tracking
  Input: Project name + current stage + bottleneck
  Output: Pipeline dashboard + progress recommendations + risk warnings

Step 6: /policy-funding-matching
  Input: Enterprise info + technology direction
  Output: Applicable policy list + compliance determination + application material framework
```

### Supply Side — Full Product R&D and Commercialization Process

```
Step 1: /product-opportunity-discovery
  Input: Technology direction / industry
  Output: Opportunity analysis report (pain point map, market gaps, competitor blind spots, differentiation entry points)

Step 2: /product-definition-planning
  Input: Product opportunity description
  Output: PRD document, MVP scope, product roadmap

Step 3: /technical-solution-demonstration
  Input: Product definition + technical constraints
  Output: Technical feasibility report (route comparison, BOM estimate, risk matrix)

Step 4: /prototype-validation-planning
  Input: Technical solution + hypotheses to be validated
  Output: EVT plan (experimental design, evaluation criteria, iteration strategy)

Step 5: /dvt-pvt-mass-production-planning
  Input: EVT results + iteration direction
  Output: DVT/PVT checklist, mass production readiness assessment, certification path

Step 6: /commercialization-strategy
  Input: Product positioning + target market
  Output: GTM strategy document (pricing, channels, seed users, PMF metrics)

Step 7: /fundraising-preparation
  Input: Company stage + business data + fundraising target
  Output: BP framework, data room checklist, valuation reference, investor matching
```

### Demand-Supply Two-Side Collaboration

```
Demand side chain:
  /tech-demand-mining -> /achievement-search-matching -> /technology-valuation
  -> /deal-plan-generation -> /project-tracking -> /policy-funding-matching

Supply side chain:
  /product-opportunity-discovery -> /product-definition-planning -> /technical-solution-demonstration
  -> /prototype-validation-planning -> /dvt-pvt-mass-production-planning
  -> /commercialization-strategy -> /fundraising-preparation

Collaboration closed loop:
  1. Demand side outputs "technology demand description" for supply side to optimize product definition
  2. Supply side outputs "achievement information" and "commercialization feasibility" for demand side to refine deal-making plans
  3. Demand side "valuation range" and supply side "BOM cost" are cross-referenced to form reasonable pricing
  4. /project-tracking synchronously tracks project progress on both demand and supply sides
```

## Cross-Skill Handoff Fields

### Demand Side Internal Handoff Fields


| Field | Type | Description | Flow Direction |
| -------------- | ------- | ---------------------------------- | ---------------------------------------------- |
| `Demand ID` | string | Unique identifier generated by Tech Demand Mining | Tech Demand Mining -> Achievement Search & Matching |
| `Achievement ID` | string | Recommendation item identifier generated by Achievement Search & Matching | Achievement Search & Matching -> Technology Valuation |
| `Supplier` | object | {name, organization, contact, IP status} | Deal Plan Generation, Project Tracking |
| `Demander` | object | {name, industry, scale, technology absorption capacity} | Deal Plan Generation, Project Tracking |
| `TRL` | integer | Technology readiness level (1-9) | Achievement Search & Matching <-> Technology Valuation <-> Deal Plan Generation |
| `Match Score` | integer | Five-dimensional semantic matching score (0-100) | Achievement Search & Matching -> Deal Plan Generation |
| `Valuation Range` | string | Conservative / neutral / optimistic three-tier valuation | Technology Valuation -> Deal Plan Generation |
| `Evidence Source` | string | Data source, query date, confidence level | Required in all Skill outputs |
| `Next Step Owner` | string | Recommended next-step owner and deadline | Output by Project Tracking |
| `Risk Level` | enum | High / medium / low | Technology Valuation, Deal Plan Generation, Project Tracking |
| `Pending Information` | list | Missing fields and impact level | Output by all Skills when input is incomplete |

## Full Examples

### Example A: Demand Side — Full Solid-State Battery Technology Transfer Process

```
Step 1: /tech-demand-mining "Green Energy Power Technology, main business ternary lithium battery, planning to layout solid-state battery"
  -> Output: Demand profile (material/equipment/talent bottlenecks), technology demand list, discipline mapping

Step 2: /achievement-search-matching "Sulfide solid electrolyte, ionic conductivity >=10 mS/cm, TRL >=5"
  -> Output: Top 5 recommended achievements (HUST patent match score 92), technology route comparison matrix

Step 3: /technology-valuation "Patent No. ZL202210XXXXXX.X"
  -> Output: Technology advancement assessment, TAM/SAM/SOM estimation, pricing reference (30-50 million CNY)

Step 4: /deal-plan-generation "Supplier: HUST... Demander: Green Energy Power..."
  -> Output: Cooperation proposal (exclusive license + joint development, upfront fee 8 million CNY, milestone payments)

Step 5: /project-tracking "Project: Solid-state battery technology transfer, Stage: docking 40%, Bottleneck: cooperation model not finalized"
  -> Output: Pipeline update, progress recommendations, risk warnings

Step 6: /policy-funding-matching "Enterprise: Green Energy Power, Technology direction: solid-state battery"
  -> Output: Top 5 applicable policies (National Key R&D Program, Jiangsu Province Achievement Commercialization Special Program, etc.)
```

### Example B: Supply Side — AI Quality Inspection Machine from 0 to 1 Full Process

```
Step 1: /product-opportunity-discovery "AI industrial vision, targeting manufacturing quality inspection"
  -> Output: User pain point map, market gap (AI quality inspection machine under 50,000 CNY), competitor blind spots, differentiation strategy

Step 2: /product-definition-planning "AI quality inspection machine for SMEs, core value 7-day deployment, miss rate <1%"
  -> Output: PRD (user persona, feature tiers P0/P1/P2, technical indicators, MVP scope, roadmap)

Step 3: /technical-solution-demonstration "Based on PRD, constraints: cost <8,000 CNY, timeline 9 months"
  -> Output: System architecture, technology route comparison (RK3588 vs Jetson), BOM estimate (2,540 CNY), risk matrix

Step 4: /prototype-validation-planning "RK3588 solution, hypotheses: miss rate <1%, speed >=30 pcs/min"
  -> Output: EVT plan (hypothesis classification P0/P1, experimental design, evaluation criteria, iteration decision tree)

Step 5: /dvt-pvt-mass-production-planning "EVT passed, iteration: optimize installation process"
  -> Output: DVT/PVT checklist, mass production readiness assessment (current 3.5), certification path (CCC/CE)

Step 6: /commercialization-strategy "Product positioning: AI quality inspection for SMEs, Target: Yangtze River Delta"
  -> Output: GTM strategy (pricing from 19,800 CNY, channels, 20 seed users, PMF metrics system)

Step 7: /fundraising-preparation "Pre-A round, no revenue yet, fundraising 8 million CNY"
  -> Output: BP framework, data room checklist, valuation reference (40-50 million CNY), fundraising timeline 16 weeks
```
