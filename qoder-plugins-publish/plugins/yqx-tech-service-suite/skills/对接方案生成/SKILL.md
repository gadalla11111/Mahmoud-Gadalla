# 科技服务助手套件使用说明

## 概述

科技服务助手是一套贯通需求侧与供给侧的全流程工具套件，覆盖技术转移、产品研发与商业化的完整闭环。本套件包含13个专业技能，分为需求侧6个技能和供给侧7个技能，支持独立调用或链路协同。

## 使用流程

### 需求侧链路（技术引进方）

按顺序调用以下技能，完成从需求识别到政策匹配的全流程：

1. **技术需求挖掘** → 识别企业技术瓶颈，输出需求画像与学科映射
2. **成果搜索与匹配** → 多源检索匹配科技成果、专利与研发团队
3. **技术价值评估** → 评估技术商业化可行性，输出定价参考
4. **对接方案生成** → 生成技术转移合作方案书，含交易结构与里程碑
5. **项目跟进管理** → 全生命周期Pipeline管理，风险预警与行动清单
6. **政策与资金匹配** → 智能匹配可申报政策、基金与补贴

### 供给侧链路（技术输出方）

按顺序调用以下技能，完成从产品机会到融资准备的全流程：

1. **产品机会发现** → 运用设计思维发现产品机会，输出痛点地图
2. **产品定义与规划** → 输出PRD文档、MVP范围与产品路线图
3. **技术方案论证** → 技术可行性分析，含路线对比、BOM估算与风险矩阵
4. **原型验证规划** → 输出EVT计划书，含测试方案与迭代策略
5. **设计验证与量产规划** → DVT/PVT检查清单、量产准备度评估
6. **商业化策略** → GTM策略文档，含定价、渠道与PMF验证指标
7. **融资准备** → BP框架、数据房清单、估值参考与投资人匹配

### 供需协同

- 需求侧输出的"技术需求描述"可供给供给侧优化产品定义
- 供给侧输出的"成果信息"与"商业化可行性"可供给需求侧完善对接方案
- 需求侧"估值区间"与供给侧"BOM成本"双向参考，形成合理定价
- **项目跟进管理**同步跟踪供需双方项目推进状态

## 输入输出规范

### 通用要求

- 所有输出须标注数据来源、查询日期及置信度
- 不确定性信息须明确说明局限性
- 技术需求描述须具体可量化，避免空泛表述
- 执行五维合规校验：政治合规、政策合规、数据合规、广告宣传合规、隐私合规

## Use Cases

- 基于供方成果与需方企业信息，生成技术转移合作方案书
- 为技术转让、许可、合作开发、技术入股等场景匹配合适的合作模式
- 设计交易结构（定价、支付安排、权利义务）与里程碑计划
- 识别技术转移过程中的风险点并提供防控措施

## Example

```bash
# 输入供方与需方信息生成合作方案
供方：某高校专利"高性能复合材料制备技术"（TRL7），知识产权归高校所有
需方：某汽车零部件制造企业，年营收5亿元，计划布局轻量化材料
请生成技术转移合作方案书
```

## Output

- 技术转移合作方案书（Markdown格式）
- 包含：合作模式推荐（首选/次选/备选）、交易结构草案、里程碑计划（五阶段）、风险识别与防控、合规与审批要求



- 合作模式匹配：基于供需特征从六种模式中推荐首选/次选/备选，包括所有权转让、独占许可、排他许可、普通许可、合作开发、技术入股。详见 [合作模式详解](references/合作模式详解.md)
- 交易结构设计涵盖定价机制（固定/里程碑/提成/混合）、支付安排、权利义务与改进成果归属
- 里程碑计划按签约→交割→验证→量产→市场五阶段设计，每阶段明确交付物、验收标准、付款节点。详见 [输出模板](references/输出模板.md)
- 风险识别覆盖技术有效性、知识产权瑕疵、人员流动、市场变化、税务合规五大风险点
- 五维合规校验：①政治合规（不涉及敏感技术违规转移）；②政策合规（引用《民法典》《专利法》《促进科技成果转化法》等，不编造法条）；③数据合规（交易对价参考公开案例，不虚构数据）；④广告宣传合规（不承诺收益，不使用"稳赚""包成功"等表述）；⑤隐私合规（不泄露双方未公开商业信息、个人隐私）。校验不通过则打回修改
- 涉及国有资产的技术转移需履行评估与审批程序
- 跨境技术转移需符合出口管制与外资安全审查要求
- 高校院所成果转化需符合其内部管理规定
- 本方案为框架性建议，正式协议需经双方法务/律师审核
