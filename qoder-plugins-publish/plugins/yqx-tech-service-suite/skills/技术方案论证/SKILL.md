# 科技服务助手套件使用说明

## 概述

科技服务助手是一套贯通需求侧与供给侧的全流程工具套件，覆盖技术转移、产品研发与商业化的完整闭环。本套件包含13个专业技能，分为需求侧6个技能和供给侧7个技能，支持独立调用或链路协同。

## 使用流程

### 需求侧链路（技术引进方）

按顺序调用以下技能，完成从需求识别到政策匹配的全流程：

1. **技术需求挖掘** → 识别企业技术瓶颈，输出需求画像与学科映射
2. **成果搜索与匹配** → 多源检索匹配科技成果、专利与研发团队
3. **技术价值评估** → 评估技术商业化可行性，输出定价参考
4. **对接方案生成** → 生成技术转移合作方案书，含交易结构与里程碑
5. **项目跟进管理** → 全生命周期Pipeline管理，风险预警与行动清单
6. **政策与资金匹配** → 智能匹配可申报政策、基金与补贴

### 供给侧链路（技术输出方）

按顺序调用以下技能，完成从产品机会到融资准备的全流程：

1. **产品机会发现** → 运用设计思维发现产品机会，输出痛点地图
2. **产品定义与规划** → 输出PRD文档、MVP范围与产品路线图
3. **技术方案论证** → 技术可行性分析，含路线对比、BOM估算与风险矩阵
4. **原型验证规划** → 输出EVT计划书，含测试方案与迭代策略
5. **设计验证与量产规划** → DVT/PVT检查清单、量产准备度评估
6. **商业化策略** → GTM策略文档，含定价、渠道与PMF验证指标
7. **融资准备** → BP框架、数据房清单、估值参考与投资人匹配

### 供需协同

- 需求侧输出的"技术需求描述"可供给供给侧优化产品定义
- 供给侧输出的"成果信息"与"商业化可行性"可供给需求侧完善对接方案
- 需求侧"估值区间"与供给侧"BOM成本"双向参考，形成合理定价
- **项目跟进管理**同步跟踪供需双方项目推进状态

## 输入输出规范

### 通用要求

- 所有输出须标注数据来源、查询日期及置信度
- 不确定性信息须明确说明局限性
- 技术需求描述须具体可量化，避免空泛表述
- 执行五维合规校验：政治合规、政策合规、数据合规、广告宣传合规、隐私合规

## 外部系统连接规范

- **只读拉取公开数据**：可直接执行，须在输出中标注数据来源、查询日期与数据性质
- **涉及写入/推送/同步操作**：必须向用户展示目标系统、字段映射、权限要求、影响范围，并经用户显式确认后方可执行
- **涉及商业数据库/付费接口/私有系统**：必须经用户显式确认后方可执行
- 任何操作均不得自动执行，不得超出用户明确授权范围
- 若用户明确拒绝连接，应基于已提供信息继续完成任务，不因拒绝而中断或降级任务质量

## Use Cases

- 基于PRD进行系统架构分解（L0→L1→L2）与技术路线选型

## Output

- 技术可行性分析报告（Markdown格式）
- 包含：系统架构（L0→L1→L2）、技术路线对比（关键模块≥2条）、BOM成本估算（三级）、关键路径、风险矩阵（五维）、DFM/DFA评估、供应链分析



- **核心方法论**：系统思维——架构分解（MECE）→关键路径识别→风险矩阵评估
- **系统架构**：按L0整机→L1子系统→L2模块层级分解，明确功能边界、接口定义、性能要求与技术约束
- **技术路线对比**：关键模块至少提供2条路线，对比维度含技术原理、性能上限、成熟度、BOM成本、供应链可得性、专利壁垒、扩展性
- **BOM成本**：分三级估算，需区分NRE（一次性工程费用）与单位成本；价格基于公开市场数据或供应商报价
- **关键路径**：识别长周期器件（定制芯片/模具/特种材料）、技术验证点与外部依赖，输出路径图与缓冲时间建议
- **风险矩阵**：五维覆盖——技术风险、成本风险、供应链风险、工艺风险、进度风险；按概率×影响评定风险等级并制定缓解措施
- **DFM/DFA评估**：审视零件数量、装配方向、公差要求、标准件使用率、专用工装/夹具需求
- **供应链分析**：需考虑地缘政治与贸易政策影响，核心器件建议双源采购或国产替代
- **参考文档**：

  - [references/系统架构方法.md](references/系统架构方法.md) — 架构分解原则、接口定义方法、技术路线对比维度、BOM估算方法、关键路径法（CPM）
  - [references/风险评估模板.md](references/风险评估模板.md) — 风险矩阵模板、DFM/DFA检查清单、供应链风险评估框架
- **五维合规校验**：

  1. 政治合规：技术选型不涉及受限技术出口
  2. 政策合规：符合行业技术标准与安全规范
  3. 数据合规：BOM价格基于公开市场数据或供应商报价
  4. 广告宣传合规：成本估算标注精度范围，不做绝对承诺
  5. 隐私合规：供应链分析不涉及供应商商业秘密
- **输入兜底**：若技术约束未提供（成本/工期/性能上限缺失），基于行业基准假设进行估算，并在报告中明确标注"假设条件"与"建议补充确认项"，避免用户误将假设当作承诺
- **外部系统连接（仅草案）**：区分操作类型与数据性质，差异化处理：

  - **只读拉取公开数据**（如政府公开网站、国家知识产权局公开数据库、科技部/工信部公开政策接口、学术期刊开放获取数据、法院判决公开信息等）：可直接执行，无需显式确认，但须在输出中标注数据来源、查询日期与数据性质（公开数据）
  - **涉及写入/推送/同步操作**（如写入Notion/飞书/CRM、推送广告后台、同步项目管理工具/实验室排期系统、同步Jira等）：必须向用户展示目标系统、字段映射、权限要求、影响范围，并经用户显式确认后方可执行
  - **涉及商业数据库/付费接口/私有系统**（如企查查、IT桔子、Statista、Sensor Tower、立创商城API、腾讯广告API、认证机构付费接口等）：必须经用户显式确认后方可执行
  - 任何操作均不得自动执行，不得超出用户明确授权范围
  - **拒绝执行兜底**（适用于需确认场景）：若用户明确拒绝连接或表示不需要外部数据（如回复"不需要""不用了""跳过"等），skill应回应"已收到，将基于您已提供的信息继续完成分析，不连接外部系统。"并正常输出本skill核心交付物，不因用户拒绝而中断或降级整体任务质量。
  - **统一确认话术模板**（适用于需确认场景）："拟执行以下外部系统操作：【操作简述，如'从XX系统拉取YY数据'/'向XX系统写入YY数据'】。目标系统：【系统名称】；涉及字段：【字段映射列表】；权限要求：【所需授权说明】；影响范围：【数据范围与风险说明】。请确认是否执行？（回复'确认'以继续，或回复'不需要'以跳过）"
  - **Session内确认复用**（适用于需确认场景）：在同一对话session中，若用户此前已确认连接某外部系统且操作类型（拉取/写入/推送）与字段映射未发生变更，再次涉及该系统的确认时可简化为："是否继续使用【系统名称】执行【操作简述】？（回复'确认'继续，或'不需要'跳过）"。若操作类型、字段映射或权限要求发生变更，或进入新session，仍需按完整模板重新确认。
