# EVT测试方法

## 一、假设分类与优先级

### 1.1 假设分类体系

| 类型 | 定义 | 典型示例 | 验证方式 |
|------|------|----------|----------|
| **技术假设** | 关于技术性能、可行性、集成度的假设 | 某传感器精度可达0.1mm；电池续航可达8小时 | 台架测试、原型测试、环境测试 |
| **用户假设** | 关于用户需求、行为、付费意愿的假设 | 用户愿为节省30%时间付费；老年用户能独立完成操作 | 用户访谈、可用性测试、A/B测试 |
| **市场假设** | 关于市场规模、渠道、竞争格局的假设 | 目标细分市场年增长率>15%；某渠道可覆盖60%目标客户 | 市场调研、渠道测试、销售试点 |
| **商业假设** | 关于成本结构、定价、单位经济的假设 | 量产BOM可降至目标价；订阅续费率>70% | 成本测算、定价实验、小批量试销 |
| **合规假设** | 关于认证、标准、法规符合性的假设 | 产品可通过EMC Class B认证；符合某行业安全标准 | 预测试、第三方检测、法规咨询 |

### 1.2 优先级定义

| 优先级 | 名称 | 定义 | 失败后果 | 资源分配建议 |
|--------|------|------|----------|--------------|
| **P0** | Kill-Factor | 若失败则项目必须终止的假设 | 项目终止或方向根本性调整 | 优先验证，分配最优资源 |
| **P1** | 重大假设 | 影响产品定义、核心功能或商业模式的假设 | 产品定义重大调整 | 早期验证，与P0并行或紧随其后 |
| **P2** | 次要假设 | 影响体验、成本或运营效率的假设 | 局部优化或调整 | 在P0/P1验证通过后进行 |
| **P3** | 优化假设 | 影响细节体验或边际收益的假设 | 微调或放弃 | 产品化阶段验证 |

### 1.3 假设登记册模板

```markdown
| 假设ID | 假设类型 | 优先级 | 假设描述 | 当前置信度 | 验证状态 | 验证方式 | 责任人 |
|--------|----------|--------|----------|------------|----------|----------|--------|
| H001 | 技术 | P0 | 核心算法在目标场景下准确率>=95% | 中 | 待验证 | 台架测试 | |
| H002 | 用户 | P1 | 目标用户可在无培训情况下5分钟上手 | 低 | 待验证 | 可用性测试 | |
| H003 | 商业 | P2 | 量产BOM较当前估算可下降15% | 中 | 待验证 | 供应商询价 | |
```

## 二、实验设计原则

### 2.1 科学实验三原则

#### 对照（Control）
- 每个实验需设置对照组或基准线
- 对照条件应与实验组除待测变量外保持一致
- 对照目的：排除无关因素干扰，确认观测差异确实来自待测变量

#### 随机（Randomization）
- 样本分配应随机化，避免系统性偏差
- 测试顺序应随机化，避免时间效应（如学习效应、疲劳效应）
- 随机方法：随机数表、计算机随机分配、分层随机

#### 重复（Replication）
- 实验需有足够的重复次数以保证结果可靠性
- 重复包括：同条件下多次测量、多个样本、多批次验证
- 重复目的：估计实验误差、提高统计功效

### 2.2 实验设计类型选择

| 设计类型 | 适用场景 | 优点 | 注意事项 |
|----------|----------|------|----------|
| 完全随机设计 | 样本同质、条件单一 | 简单直观 | 样本量需足够大 |
| 配对设计 | 样本间存在配对关系 | 减少个体差异影响 | 配对依据需合理 |
| 区组设计 | 已知干扰因素 | 控制已知变异 | 区组划分需科学 |
| 正交实验 | 多因素多水平 | 用较少实验次数筛选关键因子 | 仅分析主效应与部分交互 |
| 全因子实验 | 因素少、需全面分析交互 | 信息完整 | 实验次数随因素指数增长 |

### 2.3 最小验证实验（MVE）设计

**原则**：用最少资源获取最大学习价值

**设计步骤**：
1. 明确待验证假设（一次只验证一个核心假设）
2. 定义最小可观测差异（什么结果算"有效"）
3. 选择最简单的实验方式（仿真>台架>原型>用户）
4. 估算所需样本量
5. 设计对照与记录格式
6. 预设停止规则（何时终止或扩大实验）

## 三、样本量估算方法

### 3.1 基本原则
样本量需同时满足统计显著性与实际可行性，过小则结论不可靠，过大则浪费资源。

### 3.2 连续变量样本量估算（均值比较）

**已知条件**：
- 期望检测的最小差异 δ（delta）
- 历史或预实验估计的标准差 σ（sigma）
- 显著性水平 α（通常取0.05）
- 统计功效 1-β（通常取0.80或0.90）

**估算公式（每组样本量）**：

```
n ≈ 2 × ((Z_(1-α/2) + Z_(1-β)) × σ / δ)^2
```

**常用Z值**：
- Z_(0.975) = 1.96（α=0.05, 双侧）
- Z_(0.80) = 0.84（功效80%）
- Z_(0.90) = 1.28（功效90%）

**示例**：
希望检测两组均值差异 δ=0.5，估计标准差 σ=1.0，α=0.05，功效80%

```
n ≈ 2 × ((1.96 + 0.84) × 1.0 / 0.5)^2
n ≈ 2 × (2.8 / 0.5)^2
n ≈ 2 × 31.36 ≈ 63
```

每组约需63个样本，总计126个。

### 3.3 比例变量样本量估算（合格率/转化率）

**已知条件**：
- 基准比例 p1
- 期望比例 p2
- 显著性水平 α
- 统计功效 1-β

**估算公式**：

```
n ≈ (Z_(1-α/2) × sqrt(2×p̄×(1-p̄)) + Z_(1-β) × sqrt(p1×(1-p1)+p2×(1-p2)))^2 / (p1-p2)^2

其中 p̄ = (p1 + p2) / 2
```

**快速参考表（α=0.05, 功效=80%）**：

| 基准比例 | 期望比例 | 每组样本量 |
|----------|----------|------------|
| 50% | 60% | 约200 |
| 50% | 70% | 约50 |
| 80% | 90% | 约75 |
| 90% | 95% | 约200 |

### 3.4 实际调整因素

| 因素 | 调整方法 |
|------|----------|
| 预期脱落/失效 | 样本量 ÷ (1 - 脱落率) |
| 多组比较 | 使用Bonferroni校正或增加样本量 |
| 有限总体 | 若总体N<10n，使用有限总体校正 |
| 资源限制 | 降低功效至80%、增大可接受差异、或采用序贯检验 |

## 四、统计显著性判断

### 4.1 基本概念

| 术语 | 含义 | 常用阈值 |
|------|------|----------|
| **P值** | 在原假设为真时，观察到当前或更极端结果的概率 | P<0.05为显著 |
| **置信区间（CI）** | 估计值的不确定范围，95%CI表示重复实验95%会落入此区间 | 不包含零/无效值为显著 |
| **统计功效** | 正确拒绝错误原假设的概率 | 通常要求≥80% |
| **效应量（Effect Size）** | 实际差异的大小，不受样本量影响 | 需结合实际判断 |

### 4.2 判断流程

```
实验数据收集完成
    |
    v
数据清洗与异常值检查
    |
    v
选择适当统计检验
    |
    v
计算P值与置信区间
    |
    v
P < 0.05 ?
    |--否--> 结果不显著，无法拒绝原假设
    |--是--> 结果统计显著
                |
                v
            效应量是否实际有意义？
                |--否--> 统计显著但无实际价值
                |--是--> 结果可靠且有意义
                            |
                            v
                        记录结论与置信度
```

### 4.3 常用统计检验选择

| 数据类型 | 场景 | 检验方法 |
|----------|------|----------|
| 连续变量，单组 vs 目标值 | 验证指标是否达标 | 单样本t检验 |
| 连续变量，两组比较 | 路线A vs 路线B | 独立样本t检验（或Mann-Whitney U） |
| 连续变量，配对比较 | 改进前后对比 | 配对t检验（或Wilcoxon符号秩） |
| 比例变量 | 合格率/转化率比较 | 卡方检验（或Fisher精确检验） |
| 多组比较 | 三种方案对比 | 单因素方差分析（ANOVA） |

### 4.4 注意事项
- 统计显著 ≠ 实际重要：大样本量可能使微小差异"显著"
- 不显著 ≠ 无差异：可能样本量不足或变异过大
- 避免"P值 hacking"：不事后改变假设或选择数据子集
- 报告效应量：除P值外，报告实际差异大小

## 五、迭代决策树

### 5.1 验证结果分类

| 结果类别 | 定义 | 判定标准 |
|----------|------|----------|
| **通过** | 假设得到充分支持 | 所有关键指标达到预设成功标准，统计显著 |
| **有条件通过** | 假设部分支持，存在可解释偏差 | 主要指标达标，次要指标偏差在可接受范围，根因明确 |
| **不通过** | 假设未得到支持 | 关键指标未达标，或存在系统性问题 |

### 5.2 迭代决策树

```
                    验证结果
                      |
        +-------------+-------------+
        |             |             |
      通过        有条件通过      不通过
        |             |             |
        v             v             v
   进入下一阶段   根因是否明确？   根因分析
        |        +------+------+      |
        |        |             |      v
        |      明确          不明  假设本身不成立？
        |        |             |   +-------+-------+
        |        v             v   |               |
        |   设计修改        补充实验  是             否
        |        |             |   |               |
        |        v             v   v               v
        |   局部重测      深入分析   终止路径      实现方式问题
        |        |             |   启动备选方案    |
        |        v             v                   v
        |   重新评估      修正假设               技术方案调整
        |        |             |                   |
        +--------+-------------+-------------------+
                 |
                 v
           更新假设登记册
                 |
                 v
           规划下一轮验证
```

### 5.3 各分支处理要点

**通过分支**：
- 记录验证条件与环境，确保可复现
- 更新假设置信度为"高"
- 释放该假设占用的验证资源
- 检查是否解锁后续依赖假设的验证

**有条件通过分支**：
- 明确记录"条件"内容与边界
- 制定监控计划，在后续阶段持续跟踪
- 评估条件对产品定义的影响
- 如条件涉及安全风险，升级至P0处理

**不通过分支 — 假设不成立**：
- 召开评审会，确认非实验误差导致
- 回溯至产品定义或技术方案阶段
- 启动预先准备的备选方案
- 更新项目风险登记册
- 如涉及重大方向调整，同步利益相关方

**不通过分支 — 实现方式问题**：
- 区分是设计缺陷还是执行偏差
- 设计缺陷：修改方案，重新验证
- 执行偏差：优化工艺/流程，重新测试
- 设定重测次数上限（通常不超过3轮）

### 5.4 迭代管理原则
1. **快速闭环**：单轮验证周期尽量压缩至2-4周
2. **假设前置**：每轮迭代前明确本轮要更新的假设
3. **知识沉淀**：每轮验证无论结果均需产出书面报告
4. **资源止损**：对连续2轮不通过的P0假设，强制启动备选方案评审
