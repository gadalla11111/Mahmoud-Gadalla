# 科技服务助手套件使用说明

## 概述

科技服务助手是一套贯通需求侧与供给侧的全流程工具套件，覆盖技术转移、产品研发与商业化的完整闭环。本套件包含13个专业技能，分为需求侧6个技能和供给侧7个技能，支持独立调用或链路协同。

## 使用流程

### 需求侧链路（技术引进方）

按顺序调用以下技能，完成从需求识别到政策匹配的全流程：

1. **技术需求挖掘** → 识别企业技术瓶颈，输出需求画像与学科映射
2. **成果搜索与匹配** → 多源检索匹配科技成果、专利与研发团队
3. **技术价值评估** → 评估技术商业化可行性，输出定价参考
4. **对接方案生成** → 生成技术转移合作方案书，含交易结构与里程碑
5. **项目跟进管理** → 全生命周期Pipeline管理，风险预警与行动清单
6. **政策与资金匹配** → 智能匹配可申报政策、基金与补贴

### 供给侧链路（技术输出方）

按顺序调用以下技能，完成从产品机会到融资准备的全流程：

1. **产品机会发现** → 运用设计思维发现产品机会，输出痛点地图
2. **产品定义与规划** → 输出PRD文档、MVP范围与产品路线图
3. **技术方案论证** → 技术可行性分析，含路线对比、BOM估算与风险矩阵
4. **原型验证规划** → 输出EVT计划书，含测试方案与迭代策略
5. **设计验证与量产规划** → DVT/PVT检查清单、量产准备度评估
6. **商业化策略** → GTM策略文档，含定价、渠道与PMF验证指标
7. **融资准备** → BP框架、数据房清单、估值参考与投资人匹配

### 供需协同

- 需求侧输出的"技术需求描述"可供给供给侧优化产品定义
- 供给侧输出的"成果信息"与"商业化可行性"可供给需求侧完善对接方案
- 需求侧"估值区间"与供给侧"BOM成本"双向参考，形成合理定价
- **项目跟进管理**同步跟踪供需双方项目推进状态

## 输入输出规范

## Output

- 技术价值评估报告（Markdown格式）
- 包含：专利基本信息、技术先进性评估（四维度）、TAM/SAM/SOM市场空间、竞品分析矩阵、转化风险评估（五类）、定价参考（三类区间）及置信度标注



- 输入识别分为三类并启动对应评估路径：专利号（专利价值评估，补充专利文本/法律状态/同族专利）、技术方案描述（技术方案评估，补充技术原理/应用场景/优势对比）、成果简介（科技成果评估，补充完成单位/成熟度/应用案例）
- 技术先进性评估涵盖四个维度：技术创新度（相对现有技术的改进程度/是否解决行业难题/是否开辟新方向）、技术壁垒（专利布局完整性/know-how复杂度/反向工程难度/替代技术威胁）、技术成熟度TRL（依据实验数据/原型状态/应用案例判定等级并说明距产业化差距）、团队持续创新能力（学术技术背景/持续产出能力/产业化经验）
- 市场空间评估采用TAM/SAM/SOM三级测算框架：TAM（总可及市场，全球容量）、SAM（可服务市场，目标区域/行业容量）、SOM（可获得市场，基于竞争格局可获取份额），并分析政策驱动、需求驱动和技术驱动三类增长因素
- 竞品分析须构建技术对比矩阵，对比维度包括核心性能指标、成本结构、成熟度、专利壁垒和市场渗透率
- 转化风险评估识别并量化五类核心风险：技术风险（放大生产可行性/可靠性/一致性）、市场风险（需求验证程度/客户接受度/替代品威胁）、政策法规风险（准入许可/标准认证/环保要求）、团队风险（核心人员稳定性/产业化能力）、知识产权风险（侵权风险/权利稳定性/无效宣告风险）
- 定价参考提供三类区间建议：许可费率（参考行业惯例如营收3%-5%）、转让对价（参考同类技术交易案例）、入股估值（参考技术作价入股市场案例）；所有定价须标注参照来源与适用条件，不给出绝对承诺，估值表述须审慎
- 五维合规校验要求：政治合规（无不当技术出口/敏感技术扩散表述）、政策合规（引用法规标准准确可查）、数据合规（市场规模数据标注来源，不编造交易案例）、广告宣传合规（不使用"价值连城""独一无二"等夸大用语）、隐私合规（不披露研发团队个人隐私和未公开技术细节）；校验不通过则打回修改，直至通过
- 所有评估结论基于公开信息与合理假设，对缺乏数据的维度明确标注"数据不足，建议补充尽调"；本评估不替代专业评估机构出具的正式报告
- **输入兜底**：若输入仅为成果简介且缺乏技术细节，输出"数据不足评估维度"清单，标注哪些维度需补充尽调资料，并提供基于公开信息的初步判断与置信度

  - **涉及商业数据库/付费接口/私有系统**（如企查查、IT桔子、Statista、Sensor Tower、立创商城API、腾讯广告API、认证机构付费接口等）：必须经用户显式确认后方可执行

  - **拒绝执行兜底**（适用于需确认场景）：若用户明确拒绝连接或表示不需要外部数据（如回复"不需要""不用了""跳过"等），skill应回应"已收到，将基于您已提供的信息继续完成分析，不连接外部系统。"并正常输出本skill核心交付物，不因用户拒绝而中断或降级整体任务质量。
