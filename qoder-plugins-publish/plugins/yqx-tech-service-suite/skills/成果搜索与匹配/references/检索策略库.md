# 检索策略库参考

## 一、专利检索策略

### 1.1 检索字段组合

| 检索目的 | 推荐字段组合 | 适用场景 |
|----------|--------------|----------|
| 技术主题初筛 | 标题 + 摘要 | 快速了解某技术方向专利总量 |
| 精准技术定位 | 摘要 + 权利要求 | 锁定具体技术方案的专利 |
| 竞争对手监控 | 申请人 + IPC分类号 | 追踪特定企业技术布局 |
| 法律状态筛选 | 权利要求 + 法律状态 | 筛选有效专利或失效专利 |

### 1.2 关键词组合策略

**策略A：核心词扩展法**

```
核心词（技术本质）
  └── 同义词扩展：考虑技术术语的多种表述
  └── 上位词扩展：将具体技术概括到更宽类别
  └── 下位词扩展：将宽泛概念细化到具体实现
  └── 近义词扩展：考虑不同学科领域的相近表述
```

**策略B：技术效果关联法**

```
技术问题/手段 + 技术效果
示例：（电机冷却 OR 电机散热）AND（效率提升 OR 温升降低）
```

**策略C：IPC分类号与关键词联用法**

- 先通过关键词检索获取代表性专利
- 提取这些专利的IPC分类号
- 用IPC分类号限定范围，再用关键词细化

### 1.3 IPC分类号选择方法

| 步骤 | 操作内容 | 工具/参考 |
|------|----------|-----------|
| 第一步 | 根据技术主题确定部（Section） | IPC分类表八大部 |
| 第二步 | 确定大类（Class）范围 | IPC分类表大类目录 |
| 第三步 | 确定小类（Subclass） | 结合技术功能或应用领域 |
| 第四步 | 确定大组/小组（Group/Subgroup） | 根据技术细节精度需求 |
| 第五步 | 验证分类号覆盖度 | 用已知相关专利反查分类号 |

**常用IPC分类号参考范围（示例）**

| 技术领域 | 常见IPC分类号范围 | 说明 |
|----------|-------------------|------|
| 新材料 | C01, C08, C22 | 无机化学、高分子、冶金 |
| 新能源 | H01M, H02J, H02K | 电池、供电、电机 |
| 智能制造 | B23, B25, G05B | 机床、手动工具、控制系统 |
| 信息技术 | G06F, G06N, H04L | 计算、AI、通信 |
| 生物医药 | A61K, C07, C12N | 医药、有机化学、微生物 |

## 二、论文检索策略

### 2.1 数据库选择指南

| 数据库类型 | 代表平台 | 适用场景 | 特点 |
|------------|----------|----------|------|
| 综合学术 | Web of Science, Scopus | 跨学科检索、引文分析 | 覆盖广、引文数据完整 |
| 工程索引 | Ei Compendex | 工程技术领域 | 工程应用导向、会议论文收录 |
| 中文综合 | CNKI, 万方, 维普 | 中文学术成果 | 学位论文、期刊覆盖全 |
| 预印本 | arXiv, SSRN, 中国预印本 | 获取最新未发表成果 | 时效性强、未经同行评审 |
| 专利文献 | 各国专利局数据库 | 技术细节检索 | 技术方案公开充分 |

### 2.2 引用分析方法

**前向引用分析**：从一篇核心论文出发，查找引用该论文的后续研究
- 用途：追踪技术演进路线、发现最新改进方案
- 操作：在Web of Science/Google Scholar中查看"Cited by"

**后向引用分析**：从一篇论文的参考文献中挖掘基础技术
- 用途：追溯技术源头、理解理论基础
- 操作：阅读参考文献列表，筛选高被引经典文献

**共被引分析**：两篇文献共同被多篇后续文献引用，则主题关联度高
- 用途：发现技术集群、识别研究前沿
- 操作：借助CiteSpace、VOSviewer等工具实现

### 2.3 论文筛选排序规则

| 排序维度 | 优先级 | 说明 |
|----------|--------|------|
| 相关性 | 首要 | 标题/摘要/关键词与需求的匹配程度 |
| 期刊等级 | 次要 | SCI/SSCI分区、EI收录、核心期刊目录 |
| 被引频次 | 参考 | 近5年被引频次反映影响力，但需排除自引 |
| 发表时间 | 参考 | 近3年论文代表前沿，经典论文提供理论基础 |
| 研究团队 | 参考 | 该团队是否持续产出相关方向成果 |

## 三、成果库检索策略

### 3.1 成果库类型与检索要点

| 成果库类型 | 代表平台 | 检索字段 | 注意事项 |
|------------|----------|----------|----------|
| 国家科技成果库 | 国家科技成果网 | 成果名称、完成单位、关键词 | 注意登记时间与实际完成时间的差异 |
| 技术交易市场 | 各地技术交易所、科易网 | 技术领域、应用方向、成熟度 | 关注转化状态与交易意向 |
| 高校/院所成果库 | 各高校技术转移中心 | 技术领域、完成人、应用案例 | 需单独联系获取详细资料 |
| 企业成果/产品库 | 企业官网、产品手册 | 产品型号、技术参数 | 与学术论文可能存在信息差异 |
| 国际成果库 | NTIS, ESA, JST | 英文关键词、技术领域 | 注意技术出口管制限制 |

### 3.2 成果库检索技巧

1. **成果名称模糊检索**：技术成果名称通常带有行业特色用词，建议用核心功能词而非完整名称检索
2. **完成单位层级筛选**：国家级科研单位、重点高校、行业龙头企业的成果通常成熟度较高
3. **应用案例验证**：优先选择有明确应用案例或用户证明的成果
4. **转化状态确认**：区分"仅完成鉴定""已小试""已中试""已产业化"等不同阶段

## 四、TRL评估细则

### 4.1 TRL等级判定标准与证据要求

| TRL等级 | 等级名称 | 判定标准 | 所需证据/依据 |
|---------|----------|----------|---------------|
| 1 | 基本原理发现 | 观察到基本原理或开始提出概念 | 公开发表的论文、研究报告 |
| 2 | 技术概念形成 | 形成技术应用概念或应用方案 | 技术方案文档、可行性分析报告 |
| 3 | 关键功能验证 | 在实验室环境下验证关键功能 | 实验室测试报告、样机测试数据 |
| 4 | 实验室环境验证 | 在实验室环境中验证组件/子系统 | 实验室验证报告、性能测试数据 |
| 5 | 相关环境验证 | 在模拟相关环境中验证组件/子系统 | 模拟环境测试报告、第三方检测报告 |
| 6 | 模拟环境验证系统 | 在模拟相关环境中验证系统/原型 | 原型系统测试报告、现场试验记录 |
| 7 | 实际环境验证系统 | 在实际操作环境中验证系统原型 | 现场应用测试报告、用户试用反馈 |
| 8 | 系统完成验证 | 实际系统完成并通过测试验证 | 定型试验报告、鉴定证书、标准符合性证明 |
| 9 | 实际系统成功运行 | 系统在实际任务中成功运行 | 产业化应用证明、销售合同、用户验收报告 |

### 4.2 评估操作要点

1. **证据链完整性**：TRL判定必须有对应等级的证据支撑，不能仅凭描述推断
2. **环境真实性**：区分"实验室环境""模拟环境""实际环境"，这是判定TRL 3-7的关键
3. **系统层级**：区分"组件/子系统验证"与"系统/整机验证"，这是判定TRL 4-6的关键
4. **谨慎升级**：当证据介于两个等级之间时，按较低等级判定，并在备注中说明理由
5. **行业差异**：不同行业对同一TRL等级的实际内涵可能有差异，应参考行业惯例

### 4.3 常见误判防范

| 误判场景 | 正确判定 | 原因说明 |
|----------|----------|----------|
| 仅有论文无实验数据 | TRL 1-2 | 未验证关键功能 |
| 实验室样机未测试 | TRL 3 | 未在相关环境中验证 |
| 小试成功但未中试 | TRL 4-5 | 未在模拟实际环境中验证系统 |
| 单台设备运行良好 | TRL 7-8 | 需确认是否为定型产品 |
| 已有销售但批量小 | TRL 8-9 | 需确认是否经过规模化验证 |
