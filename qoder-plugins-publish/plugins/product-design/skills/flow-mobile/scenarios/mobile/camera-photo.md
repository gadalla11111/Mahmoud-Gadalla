# Scenario: Mobile Camera & Photo Editing（移动端相机/照片编辑）

> **研究来源**：基于对 Project Indigo（Adobe Labs）、(Not Boring) Camera、Luminar、Binsoo、Picnic、Tilt、Dropbox 等真实 iOS 产品 flow 和页面的横向分析抽象，不代表任何单一产品的具体实现。

---

## Identity

**Platform**: Mobile (H5 / React Native)
**Definition**: 以拍摄与照片后期处理为核心的移动端相机/修图 App，用户可通过全屏相机界面拍摄，然后对照片进行滤镜、参数调整、裁剪等编辑操作，最终导出并分享到社交媒体或本地相册。

**Canonical Examples**: VSCO、Lightroom Mobile、Snapseed、Camera+、(Not Boring) Camera

**Not this scenario if**:
- 产品是视频剪辑工具（CapCut / Premiere Rush，以视频为核心而非静态照片）
- 产品是 AI 图片生成工具（Midjourney / DALL-E，不是拍摄和手工修图）
- 产品是贴纸/AR 滤镜社交（Snapchat / Instagram Stories，以实时特效和发布为目的）
- 产品是矢量图形/设计工具（改用 mobile/design-tools）

---

## User Profile

| 维度 | 内容 |
|---|---|
| **主要角色** | 创意摄影用户（手机摄影 + 后期处理）/ 社交发布用户（处理后发 Instagram/小红书）|
| **核心目标** | 拍到满意的照片 → 通过滤镜/调色让照片更好看 → 分享到社交平台或保存到相册 |
| **心智模型** | 数字暗房：拍摄是「底片」，编辑是「冲印」，导出是「放大」 |
| **使用频率** | 低-中频（拍照时使用，每次会话 5-20 分钟）；图库整理偶发性（每周/每月） |
| **决策模式** | 审美驱动型：在多个滤镜效果间反复对比，追求「这张看起来最好」|
| **容错期望** | 低：编辑必须可撤销（Undo）；保存须明确区分「覆盖原图」vs「另存副本」；批量删除需二次确认 |

## IA Template

**导航模式**: Bottom Tab Bar（两种主流模式）

**模式 A：相机优先型（Camera-First，适合以拍摄为主的 App）**
```
Tab 1: 相机 / Camera  — 全屏相机界面（默认落点，App 启动即进入）
Tab 2: 图库 / Library — 照片图库（按月份分组）
Tab 3: 预设 / Presets — 预设管理/购买/探索
Tab 4: 我的 / Profile — 账户 + 订阅 + 设置
```

**模式 B：图库优先型（Library-First，适合以整理和编辑为主的 App）**
```
Tab 1: 首页 / Home    — 推荐预设 / 精选内容
Tab 2: 图库 / Library — 主图库（月份分组 + 相册）
Tab 3: 相机（中央凸起 FAB 样式）
Tab 4: 编辑 / Edits   — 编辑历史记录
Tab 5: 我的 / Profile — 账户 + 设置
```

**选择建议**: 主打「随时随拍」的 App → 模式 A；主打「精修和整理」的 App → 模式 B

**页面层级**: 2-3 级
```
L1: 相机主界面 / 图库主页
L2: 照片编辑器（全屏）/ 月份视图 / 单张图查看
L3: 导出 Modal / 裁剪工具 / 预设详情 / 相册管理
```

**权限流结构**:
```
Camera（拍摄，核心功能）:
  → App 启动时立即请求（无法等待，是 App 的核心功能）
  → 拒绝后显示说明页 + 「前往系统设置开启」Button

Photos / Photo Library（图库访问，编辑和保存）:
  → 首次进入 Library Tab 或首次保存图片时请求
  → 说明：「访问图库以选择要编辑的照片，以及将编辑结果保存回相册」
  → 注意：iOS 提供「选择照片」（Limited Access）和「完整访问」两个级别

Microphone（视频录制，可选）:
  → 首次进入视频模式时请求
```

**数据密度**: 低-中
- 相机界面：全屏预览 + 最小化 HUD（顶部工具栏 + 底部控制区）
- 编辑器：全屏图片 + 底部工具栏（横向滚动）+ 参数面板（浮层）
- 图库：月份分组缩略图 Grid（3列）/ 单张大图查看

**主要容器模式**:
| 场景 | 容器 |
|---|---|
| 编辑工具子面板（Presets / Adjust）| Bottom Sheet（自定义高度，可拖拽调整）|
| 导出选项（格式/质量/路径）| Bottom Sheet（Medium）|
| 相册创建 | Dialog（Modal，简单输入名称）|
| 添加到相册 | Bottom Sheet（相册列表选择）|
| 批量删除确认 | Dialog（不可逆操作）|
| Before/After 对比 | 长按预览区（Split View 或 Toggle）|
| 裁剪/旋转 | 全屏工具（替换编辑器主视图）|

```
┌────────────────────────────────────┐
│  ✕关闭   EXPOSURE 52   ↩  ✦  ↗   │  ← 顶部持久工具栏（撤销/自动/分享）
├────────────────────────────────────┤
│                                    │
│                                    │
│         [全屏照片预览]             │  ← 实时预览区（手势缩放/对比）

├────────────────────────────────────┤

├────────────────────────────────────┤

```

## 该场景独有的 IA/UX 决策

1. **相机界面必须最小化 HUD，全屏预览区不能被工具栏遮挡** — 相机 App 的核心价值是「所见即所得」——用户需要在取景时看到尽可能完整、真实的画面，才能做出正确的构图判断。如果顶部工具栏（Flash / Grid / 计时器）用实色背景占据 50pt 高度，不仅遮挡画面内容，也破坏了「拿起手机即是取景器」的沉浸感。正确做法：顶部工具栏小图标透明叠加于预览区上方；底部控制区（快门 / 镜头切换 / 缩略图）用半透明暗色背景，预览区延伸至安全区域——Project Indigo（flow 11763）和 (Not Boring) Camera（flow 7682）均采用最小化 HUD 布局。

2. **照片编辑必须全程非破坏性（Non-Destructive），导出时明确区分「覆盖原图」vs「另存副本」** — 修图是一个高度「反复尝试」的过程，用户在调色过程中可能修改十几次才满意，中途可能想回到原始状态对比。如果任何调整直接写入原图，用户将无法恢复失误操作，产生强烈的不安全感。正确做法：编辑器内所有操作均为非破坏性（可随时 Undo / Redo / 重置），导出时明确提供「保存到相册（覆盖原图）」和「另存为副本（保留原图）」两个选项，并在 Modal 中清晰说明区别——Luminar（flow 5099，6 屏）是此模式的完整实现，两个选项的名称和说明文字都经过精心设计以防止误操作。

3. **参数调整必须用「浮动药丸标签」实时展示参数名+数值，不能只显示滑块位置** — 用户在调整曝光（Exposure）时，如果只看到一个滑块在水平移动，无法准确判断「当前调整量是多少」，也无法在多次会话间重现相同的调整效果。正确做法：每当用户激活某个参数（点击参数行或拖动滑块），在预览图上方出现浮动药丸标签（如「EXPOSURE +52」），实时跟随数值变化——Luminar（flow 5078）是此模式的标准实现，浮动标签出现在图片中央偏上，不遮挡主体，调整完成后自动淡出。

4. **预设（Preset / Filter）必须在缩略图选择时就实时预览效果，不能先选再预览** — 用户选择滤镜时的核心需求是「在选之前就看到效果」，而不是「先选中，再看结果，不满意再换下一个」。点击后才更新预览的模式每次操作都有延迟感，用户需要在心里维护「当前预设对应哪个效果」的映射，认知负担高。正确做法：预设缩略图本身就显示当前照片应用该预设后的效果（小图实时渲染），点击后主预览区平滑切换——这样用户在横向滑动预设列表时就能看到所有效果的对比，选择成本极低。Luminar（flow 5092）和 Binsoo（flow 6792）均实现了缩略图即时预览。

5. **图库整理必须提供「Compare Mode（对比整理模式）」，解决「相似照片太多不知道留哪张」的高频痛点** — 手机摄影的典型问题是「连拍 10 张差不多的照片，最后每张都留着」，导致图库迅速膨胀（1万+张）。普通的单张浏览无法高效解决相似照片的取舍问题——用户需要同时看到两张对比才能做出判断。正确做法：提供「月份对比整理模式」，将同月照片两两配对展示（全屏并列或叠层卡片），每对提供「保留」/ 「删除/归档」/ 「收藏」三个快速操作——Picnic（flow 11499，17 屏）是此模式的完整实现，底部缩略图条实时显示每张的决策状态（绿/红点）。

## Canonical Flows

> 以下 flow 基于 7 个真实产品样本横向分析抽象。括号内标注「行业共识」表示多个产品采用相同模式。

---

### Flow 1: 拍摄（Camera Capture）

**在此场景的特殊性**: 相机 App 的拍摄界面与 IM 类 App 的「附件拍照」有根本差异——相机 App 的拍摄是主功能，界面设计围绕「构图 + 曝光控制 + 快速拍摄」展开，而非「拍一张附件」。Project Indigo（flow 11763）展示了专业参数 HUD（直方图 / 快门 / ISO / 曝光读数）；(Not Boring) Camera（flow 7686）实现了「相机预览画布即预设选择器」的创新设计——左右滑动直接切换预设效果，无需打开单独的面板。拍摄后缩略图即时出现在左下角（Tilt flow 3090），让用户确认成果不需要离开相机界面。

**行业共识**：Project Indigo（flow 11763/11755）、(Not Boring) Camera（flow 7682）确认：全屏预览 + 最小化顶部工具栏 + 底部三分布局（缩略图 | 快门 | 镜头切换）；拍摄后缩略图即时出现在快门左侧。

**Entry**: App 启动（相机优先模式）/ 点击 Tab Bar 相机图标

```
Screen 1: 相机主界面（Camera View）
  主操作: 构图 + 拍摄
  关键组件:
    - 全屏预览区（Live View，占满屏幕）:
        点击对焦（黄色方框，点击位置出现，0.3s 后淡出）
        双指捏合/展开: 变焦
        上下滑动: 曝光补偿（太阳图标随手势移动）
        [可选] 网格线（Grid，Rule of Thirds，设置中开关）
    - 顶部工具栏（透明叠加，小图标）:
        Button（⚡ 闪光灯，三态: 自动/开/关）
        Button（⊞ 网格）
        Button（⏱ 计时器，0/3/10秒）
        Button（HDR / RAW 切换，如支持）
        [专业模式] 直方图（实时亮度分布小图）
    - 中央控制区（专业模式）:
        快门速度 / ISO / 曝光偏差读数横排（点击可手动调整）
    - 底部控制区（半透明暗色背景）:
        左侧: 最近拍摄缩略图（圆角正方形，60pt，点击进入 Screen 2 预览）
        中央: 快门按钮（大圆形白色，按下 = 拍照；长按 = 录视频）
            拍摄计数徽章（「0/10」，引导完成目标数量，部分 App 有）
        右侧: 镜头切换按钮（「0.5x / 1x / 2x」文字 Toggle）
    - 模式选择行（底部控制区上方，横向可滑动）:
        「延时 / 视频 / 照片 / 人像 / Pro」（中央当前模式高亮）
    - [可选，(Not Boring) Camera 模式] 预设选择器（非独立面板）:
        整个相机预览画布左右滑动 = 切换预设效果
        中央圆形「锁定」按钮（锁定当前预设 → 按钮显示锁图标）
        右侧强度轮（蓝色侧轮，控制预设强度）
  → 短按快门: 拍照，快门动画（屏幕白色闪烁）→ 缩略图更新到左下角
  → 拍照后（Screen 1 状态不变）: 用户可继续拍摄，无需确认
  → 点击左下角缩略图: Screen 2（拍摄后预览）
  → 长按快门: 进入录视频状态（快门变为红色录制按钮）

Screen 2: 拍摄后预览（Shot Preview）
  主操作: 确认照片质量，决定保留或删除，或立即分享
  容器: 全屏图片查看，底部固定操作栏
  关键组件:
    - 全屏照片展示（支持双指缩放查看细节）
    - 文件信息（右上角，如「SUPER RAW」/ 「JPEG」格式徽章）
    - [专业模式] AI 处理进度（「Processing...」小圆圈，超分辨率/降噪处理中）
    - 底部操作栏（3 个按钮）:
        Button（🗑 删除，左侧，red danger，点击 → 直接删除无 Dialog，因为刚拍的照片）
        Button（← 返回，中央，回到相机继续拍摄）
        Button（↗ 分享，右侧，进入 Flow 3 Screen 1a 导出路径）
  → 点击「← 返回」: 回到相机主界面，继续拍摄
  → 点击「🗑 删除」: 照片删除，回到相机（刚拍的直接删，无需二次确认）
  → 点击「↗ 分享」: 进入导出/分享 Modal（Flow 3）
```

**Exit State**:
- ✅ 拍摄成功：照片自动保存到系统图库，缩略图出现在快门左侧
- ❌ 相机权限被拒绝：显示说明页（「Camera 权限被拒绝，请前往设置开启」）+ Button「打开设置」
- ❌ 存储空间不足：拍摄时显示 Toast「存储空间不足，请清理照片后重试」，快门禁用

### Flow 2: 照片编辑（Filter & Adjustment）

**在此场景的特殊性**: 照片编辑是相机 App 与社交 App「添加滤镜」功能的本质差异——修图 App 提供精细的参数控制（曝光/对比/高光/阴影/白平衡等 10+ 参数），而不只是几个预设滤镜。Luminar（flow 5076-5095）是最完整的参考，展示了「底部工具 Tab 横向滚动 → 点击工具 → 参数面板上滑展开 → 浮动药丸实时数值」的完整编辑器 UX 模式。Binsoo（flow 6790，51 屏）展示了「预设探索 → 强度调整 → Recipe 保存分享」的完整预设工作流。Pinterest（flow 2534）展示了「Filter / Adjust 双 Tab 分离」的更简洁版本。

**行业共识**：Luminar（flow 5078/5092）、Binsoo（flow 6792）、Pinterest（flow 2534）均采用「底部工具 Tab 横向排列 + 选中工具后展开对应面板」的编辑器布局；浮动药丸标签（「EXPOSURE +52」）是 Luminar 等专业 App 的参数展示标准。

**Entry**: 图库 → 选中照片 → 「编辑」按钮 / 相机拍摄后 → 「编辑」CTA

```
Screen 1: 进入编辑器（Loading State）
  主操作: 等待照片加载进编辑器
  关键组件:
    - 全屏 Loading 动画（避免用户以为 App 卡死）
    - 加载完成: 直接进入 Screen 2（无需操作）

Screen 2: 编辑器主视图（Editor Main）
  主操作: 对照片进行非破坏性编辑
  关键组件:
    - 顶部工具栏（持久，透明叠加）:
        Button（✕ 关闭，左侧，返回图库）
        [中央] 浮动药丸标签（条件显示: 「EXPOSURE +52」，激活参数调整时出现，空闲时隐藏）
        Button（↩ 撤销，右侧第一个）
        Button（✦ AI 自动调整，右侧第二个）
        Button（↗ 分享/导出，右侧第三个）
    - 全屏照片预览（占据屏幕 60-70% 高度）:
        实时反映所有参数调整效果
        长按: Before/After 对比（松手恢复当前编辑状态）
        [可选] Split View（屏幕左半 = 原图，右半 = 当前编辑效果，可拖动分割线）
    - 底部工具 Tab 横向滚动栏（固定）:
        「裁剪」/ 「修复」/ 「预设」/ 「调整」/ 「遮罩」/ 「晕影」/ 「修饰」/ ...
        当前激活工具: 白色文字/图标高亮
    - [条件显示] 参数面板（点击工具 Tab 后展开，自底部上滑）:
        [预设 Presets 面板]: Screen 2a
        [调整 Adjust 面板]: Screen 2b

Screen 2a: 预设面板（Presets Panel）
  触发: 点击底部工具 Tab「预设」
  关键组件:
    - 预设分类 Chips（横向，「自然」/ 「黑白」/ 「胶片」/ 「风格」/ 「我的」）
    - 预设缩略图横向 ScrollView（每个 = 当前照片应用该预设的小图，约 60pt 宽）:
        缩略图下方: 预设名称（单行截断）
        当前激活: 白色边框高亮
        点击 = 即时切换主预览效果（无延迟）
    - 强度控制（预设缩略图下方）:
        圆形半弧旋钮（模拟物理旋钮，顺时针 = 强度增加）或水平滑块
        左标注「0」，右标注「100」，中央显示当前数值
    - [可选] 「+ 添加/购买预设」入口（横向列表末尾）
  → 滑动预设列表: 实时切换主预览效果（无需点击即可滑动预览）
  → 点击某预设: 确认选中，缩略图出现蓝色勾选角标
  → 拖动强度旋钮/滑块: 浮动药丸标签实时更新（「PRESET 75」）

Screen 2b: 参数调整面板（Adjust Panel）
  触发: 点击底部工具 Tab「调整」
  关键组件:
    - 参数列表（FlatList，每行一个参数）:
        点击参数行展开该参数的滑块（其余参数收起）
        参数行: 参数名（左，「曝光 Exposure」）+ 当前数值（右，「+52」）+ 重置 Button（双击归零）
        展开后: 全宽水平滑块（中央 0 刻度，左负右正，颜色渐变指示方向）
        **标准参数组（10个）**:
          曝光 Exposure / 对比度 Contrast / 高光 Highlights / 阴影 Shadows
          黑色 Blacks / 白色 Whites / 清晰度 Clarity / 纹理 Texture
          饱和度 Saturation / 色温 Warmth / 色调 Tint
    - 浮动药丸标签（拖动滑块时实时显示在预览图中央）: 「EXPOSURE +52」（参数名 + 数值）
    - 激活任意参数时: 顶部工具栏显示「↩ 撤销此步」
  → 拖动滑块: 主预览实时更新，浮动药丸实时刷新数值
  → 双击数值: 该参数重置为 0
  → 点击另一个参数行: 当前参数收起，新参数展开（手风琴模式）
  → 向下拖拽面板: 面板收起，回到工具 Tab 状态

Screen 3: 裁剪/旋转（Crop & Rotate，独立工具）
  触发: 点击底部工具 Tab「裁剪」
  关键组件:
    - 预览区缩小（留出裁剪 UI 空间）
    - 裁剪框（可拖拽四角/四边调整）
    - 比例约束 Chips（「自由」/ 「1:1」/ 「4:3」/ 「16:9」/ 「3:2」）
    - 旋转角度控制（底部刻度尺，左右滑动微调 ±45°）
    - Button「水平翻转」/ Button「顺时针 90°」
    - Button「完成」（右上角，应用裁剪）/ Button「取消」（左上角）
```

**Exit State**:
- ✅ 编辑完成（点击「↗」导出）: 进入 Flow 3 导出/分享
- ✅ 撤销成功（点击「↩」）: 上一步操作回滚，浮动药丸更新数值

- ↩ 关闭编辑器（无改动时）: 直接返回图库，无提示

### Flow 3: 导出与分享（Export & Share）

**在此场景的特殊性**: 照片导出是修图 App 与社交 App 的关键分叉点——社交 App 只关心「分享到哪里」，修图 App 还关心「以什么格式/质量输出」。Luminar（flow 5098/5099）是最完整的导出 UX 参考：明确区分「覆盖原图（Save to Photos）」vs「另存为副本（Save a Copy）」，并在「副本」路径下提供格式（JPEG/PNG/TIFF/HEIF）和质量（0-100%）控制——Luminar 的两个选项名称和说明文字经过精心设计以防止误操作。Picnic（flow 11496/11497）展示了「风格化导出」的差异化路径：在导出前先选择主题模板（叠加日期/地点信息），让照片更适合社交媒体发布。

**行业共识**：Luminar（flow 5098/5099）、Picnic（flow 11496）确认：Bottom Sheet 承载导出选项（非独立全屏页）；iOS 系统 Share Sheet 是最终分享节点；格式和质量控制在同一 Bottom Sheet 内完成，不分页。

**Entry**: 编辑器顶部「↗ 分享/导出」Button / 图库单张图查看底部操作栏「↗」

### Flow 4: 图库整理（Photo Library Management）

## Anti-Patterns

- **相机界面用实色背景工具栏占据屏幕顶部 50pt**：画面被遮挡，用户看不到完整的取景画面，构图受损。→ 正确做法：工具栏小图标透明叠加于相机预览上方，预览区延伸至状态栏下方，最大化取景区域（Project Indigo flow 11763 / (Not Boring) Camera flow 7682）。

- **照片编辑直接写入原图，无「另存为副本」选项**：用户保存后发现效果不满意，但原图已经被覆盖，无法恢复，产生强烈不安全感，甚至不敢使用编辑功能。→ 正确做法：导出 Modal 明确提供「覆盖原图（Save to Photos）」和「另存副本（Save a Copy）」两个选项，「覆盖原图」旁注明「原图将被替换，不可撤销」（Luminar flow 5099）。

- **预设选择需要先选中再等待效果更新（非实时预览）**：每次切换预设都有 0.5-1 秒延迟，用户需要等待才能看到效果，浏览 10 个预设需要 5-10 秒，体验极差。→ 正确做法：预设缩略图本身显示当前照片应用该预设后的效果，主预览区在用户选中时即时切换（无感延迟），Luminar（flow 5092）和 Binsoo（flow 6792）均实现了实时预览。

- **参数调整只有滑块，没有显示当前数值的标签**：用户不知道「曝光 +52」是什么概念，调整完成后无法在多次会话间重现相同的效果，调整过程缺乏精确感和控制感。→ 正确做法：拖动滑块时在预览图上方出现浮动药丸标签（「EXPOSURE +52」），松手后 1 秒自动淡出，不遮挡图片主体（Luminar flow 5078 标准实现）。

- **图库整理没有对比模式，只能一张张单独查看**：相似照片无法直接对比（「留第几张更好？」），用户最终选择「全都留着」，图库不断膨胀，App 的整理价值无法体现。→ 正确做法：提供月份「对比整理模式」，两张照片并列或叠层展示，三个快速操作（保留/删除/收藏），底部缩略图条可视化决策进度（Picnic flow 11499）。

- **批量删除无二次确认 Dialog**：用户在多选模式下误触删除，大量照片立即永久消失，极大损害用户对 App 的信任感。→ 正确做法：批量删除触发 Dialog（「删除 N 张照片？删除后无法恢复」→ 「删除」danger 样式 / 「取消」），非批量的「对比整理模式」删除操作优先使用「归档」（非破坏性，可恢复）而非直接永久删除。
