# 产品设计

设计师全流程工具箱，覆盖问题框定、设计简报、用户旅程、视觉情绪板、页面 Flow、异常态穷举、数据可视化、营销海报、动效规划与落地、设计走查、可用性测试、无障碍审计、工程 PRD 交付等 28 个核心场景。每个 Skill 内置行业方法论、条件分支和输出模板，产出可直接交付；Skill 之间通过链式 context 自动联动，无需重复输入项目背景。

> **免责声明：** 本插件辅助专业工作流程，不替代专业判断。所有输出应由具备相应资质的专业人士审核后方可用于决策。

## How It Works

```
┌─────────────────────────────────────────────────────┐
│  ALWAYS（独立工作）                                   │
│  ✓ 3 种项目入口（新建/有PRD/改版）                     │
│  ✓ 设计简报 HTML 一页纸                               │
│  ✓ 用户旅程 HTML 可视化（含情感曲线）                   │
│  ✓ 视觉情绪板 + design-tokens 推导                    │
│  ✓ Web/Mobile 多屏 Flow + 实际代码                    │
│  ✓ 异常态 6 类 25 子类型穷举                           │
│  ✓ 动效规划（Disney 12 原则）+ 代码落地                │
│  ✓ WCAG 2.1 完整合规审计                              │
│  ✓ 度量蓝图 + 埋点清单                                │
│  ✓ 500+ 行工程 PRD                                   │
│  ✓ 链路面板自动生成（spark-output/dashboard.html）           │
├─────────────────────────────────────────────────────┤
│  SUPERCHARGED（连接工具后增强）                         │
│  + Figma → 从设计稿抽 design tokens                   │
│  + designlang → 从线上站点反向抽 19 类 tokens          │
│  + WebSearch → 竞品实时信息获取                        │
└─────────────────────────────────────────────────────┘
```

## 适用处境

先选「你处在什么处境」，再选「你做什么方向的设计」，组合定位推荐路径。

### 维度 A · 处境（你怎么工作）

| 处境 | 一句话自检 | 链路偏向 |
|---|---|---|
| **协作型设计师** | "我把 Figma 发到 PRD 群，工程师实现" | 全链路 + 重 PRD / 验收 |
| **全能型设计师** | "我一个人扛设计，但稿交给别人写码" | 全链路 + 自己补研究 / 定义 |
| **AI 原生设计师** | "我用 Cursor / Claude Code 自己 git push 上线" | 重 Board / Flow / 动效落地，跳过 PRD / 验收 |

> 另有 **设计 Lead / 总监** 横切场景：不亲自画稿，贯穿 `/设计简报` `/设计提案` `/设计度量` `/设计复盘` 四锚点。

## 快捷命令

### 01 探索

| 命令 | 说明 | 典型输入 |
|------|------|---------|
| `/问题框定` | 从模糊想法探索设计方向 | "我想做一个宠物社交平台" |
| `/读需求` | 拆解已有 PRD / 需求文档 | 上传 PRD 文档 |
| `/启发评估` | 改版项目先诊断现有产品 | "对这个 App 做体验走查" |
| `/用户研究` | 主动用户研究（访谈/可用性测试） | "做 5 人深度访谈" |

| `/工单分析` | 海量工单的高频体验问题提炼 | 上传工单 CSV |

### 02 定义

| 命令 | 说明 | 典型输入 |
|------|------|---------|
| `/设计简报` | 写一页纸设计简报（HTML） | 跑完 `/问题框定` 后自动衔接 |

| `/用户旅程` | 画用户旅程图（HTML 可视化） | "画新用户首次购买旅程" |
| `/视觉情绪板` | 基于触点情绪推 3-4 套色彩方案 | 跑完 `/用户旅程` 后自动衔接 |

### 03 设计

| 命令 | 说明 | 典型输入 |

## Skills

### 01 探索

| Skill | 说明 | 核心方法论 |
|-------|------|----------|
| Frame | 从模糊想法锚定设计方向 | 5 轮 diverge-converge + persona 画像 + HMW 卡片 |
| Scope | 拆解已有 PRD 为设计语言 | 字段提炼 + gaps 标注 + 设计触点映射 |
| Audit | 改版项目启发式评估 | Nielsen 10 原则 + 改版机会点矩阵 |
| Probe | 主动用户研究 | JTBD + persona + 情感曲线 + 痛点优先级 |
| Bench | 设计师视角竞品深拆 | 标注截图 + UX/视觉 takeaways + 视觉策略决策 |
| Signal | 海量工单体验问题提炼 | 6 类分类 + affected_pages 映射 + Top 痛点 |

### 02 定义

| Skill | 说明 | 核心方法论 |
|-------|------|----------|
| Brief | 一页纸设计简报 HTML | 策略维度 + 设计原则 + 约束条件 + 质量自检 |
| Stories | 用户故事拆解 | 故事集 + 设计触点 + 验收标准 |
| Sitemap | 站点地图 / IA 骨架 | 主导航 + 页面层级 + 信息架构 |
| Journey | 用户旅程图 HTML 可视化 | 触点 + 情感曲线 + dropout-risk 标注 |
| Board | 视觉情绪板 + design-tokens | 3-4 套色彩方案 + 字体 + 组件行 + tokens 输出 |

### 03 设计

| Skill | 说明 | 核心方法论 |
|-------|------|----------|
| Flow Web | Web 多屏 Flow + 代码 | 多屏 IA + 组件清单 + .tsx 代码产出 |
| Flow Mobile | Mobile 多屏 Flow | H5/RN 适配 + 组件清单 + 代码产出 |
| Edge | 异常态穷举 | 6 类 25 子类型状态矩阵 + critical_missing |
| Chart | 数据可视化选型与规格 | 意图→数据约束→选型→视觉规格→状态→响应式六步链 |
| Avatar | 数字人 / 品牌 IP 生成 | 2 风格 × 2 主体笛卡尔积 + 4 张差异化 + ≤3 轮收敛 |
| Poster | AI 直出材质化字体海报 | 8 种材质风格 + 文字逐字校验 + 4 尺寸适配 + 可消费 design-tokens |
| Motion Plan | 动效规划 | Disney 12 原则 + Personality archetype + 元素级规范 |
| Motion Apply | 动效代码落地 | framework + scene 感知 + React Bits / CSS 双路径 |
| Pitch | 决策者设计汇报 | 6 段叙事 + 强制 Asks + 听众适配 |

### 04 验证

| Skill | 说明 | 核心方法论 |
|-------|------|----------|
| Check | 设计稿走查 | 10 类基础体检 + findings 清单 |
| Access | WCAG 2.1 完整合规审计 | 50+ 项合规检查 + 法律风险评估 |
| Test | 可用性测试 | 7 种测试类型 + Nielsen 0-4 严重度 + vertical slice 修复 |
| Metric | 设计度量蓝图 | NSM + 指标体系 + 9 项埋点清单 |

### 05 交付

| Skill | 说明 | 核心方法论 |
|-------|------|----------|
| QA | 实现还原度核查 | 9 维度差异检测 + Check 闭环 |
| PRD | 工程交付文档 | 500+ 行 PRD + 设计资产路径 + 技术约束 |
| Extract | 三模式抽 design tokens | 外部站点 / 内部代码 / Figma + 语义级 SparkDesign Diff |
| Retro | 项目上线后复盘 | 全链路回顾 + lessons learned + recommendations |
